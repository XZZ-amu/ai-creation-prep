# 人脸与安全：常见问题与风险

> 用户真人脸进入 AI 生成时，身份保真、变形容忍度、安全合规三条线上各自有什么坑。

---

## 一句话总结

人脸 AI 特效的所有问题归结为一句话：**用户要"是我又不完全是我"，平台要"不出事还得好用"，这两件事在技术上互相拉扯，在法律上互相制约。**

---

## 问题全景

八类问题，按"谁受伤"分成三个区域：

| 区域 | 谁受伤 | 问题 |
|---|---|---|
| 用户体验崩坏 | 用户本人 | 人脸不像、恐怖谷变形、多人脸混淆 |
| 他人权益侵害 | 被生成的人 | 非自愿换脸/深度伪造、名人/未成年人脸滥用 |
| 平台系统性风险 | 平台 | NSFW 绕过过滤、水印被剥离、合规风险 |

下面逐个拆开。

---

## 问题 1：人脸不像（身份保真度不够）

### 用户视角

"用了特效完全不像我，朋友根本看不出来是我。" → 用户不会保存、不会分享 → 特效的社交价值归零。

### 平台视角

渗透率和留存下降。用户试一次不像 → 对"AI 特效"整体品类建立负面印象 → 下次连试都不试。

### 技术根因

1. **人脸识别 backbone 精度不均**：ArcFace / InsightFace 在不同人种、年龄段、性别上的识别精度存在系统性差异（模型卡已承认）。深肤色用户、老年用户、化浓妆用户的 face embedding 质量更低，导致"是他"这件事在这些人群上更容易失败。

2. **ID 注入强度（scale）设低**：为了让风格更突出，产品侧把 `ip_adapter_scale` / `identitynet_scale` 调低 → 画家更自由但身份信号被稀释。在风格化特效里这是常态——风格越强，不像的概率越高。

3. **单图信息不够**：用户只传一张自拍，遮挡、侧脸、光线差 → face embedding 质量低。PhotoMaker 类多图方案能解决但用户路径过长。

4. **backbone 架构限制**：FLUX（DiT 架构）的人脸 ID 注入生态仍在追赶 SDXL；如果平台已切换到 DiT backbone，人脸工具链的成熟度还不够。

### 法律风险等级

**低**。不像不构成侵权，只构成体验差。但如果不像的概率在不同人群间不均（如深肤色用户显著更差），可能引发平等性/歧视争议。

## 问题 2：人脸变形 / 恐怖谷效应

### 用户视角

"生成出来的脸好诡异，五官位置不对 / 半边脸溶解了 / 眼睛大小不一。" → 比"不像"更糟——用户感到不适甚至恐惧。

### 平台视角

恐怖谷图片如果被用户截图传播 → "抖音 AI 生成鬼脸"类话题 → 品牌负面。

### 技术根因

1. **生成模型本身的人脸稳定性差异**：SD 1.5 人脸质量最差（底层分辨率低、人脸训练数据不够）；SDXL 较好但 CFG（Classifier-Free Guidance）调高时脸容易过饱和变形；DiT 系（FLUX、HunyuanVideo）质量最高但生态未完全成熟。

2. **身份信号与 prompt/风格信号冲突**：极端风格化（如"融化的萨尔瓦多·达利风格"）时，画家被要求同时满足"是这个人的脸"和"所有东西都是液态"，两个指令打架 → 脸的结构被部分解构 → 恐怖谷。

3. **ControlNet 姿态图不准**：如果用 OpenPose 类骨架图但自拍角度刁钻（极端侧脸、俯拍），关键点检测错误 → 脸的几何结构从一开始就错了。

4. **后处理修脸工具的副作用**：CodeFormer / GFPGAN 做人脸修复时**不保 identity**——可能把变形脸修得清晰了但更不像本人，甚至引入新的不自然感。

### 法律风险等级

**低-中**。单纯的变形不构成法律问题；但如果变形结果具有侮辱性（如把用户脸变得丑陋/怪异），用户投诉时存在"人格尊严"争议风险。

## 问题 3：多人脸混淆（张冠李戴）

### 用户视角

"一家人合影做特效，我女儿的脸跑到我老公身上去了。" → 滑稽 + 困惑 + 浪费时间。

### 平台视角

多人合影是抖音用户高频场景（情侣、闺蜜、全家福），但也是 ID 注入最容易翻车的场景。如果只支持单人 → 场景覆盖不完整 → 用户觉得"AI 特效只能自拍"。

### 技术根因

1. **主流 ID 注入方案只支持单脸**：InstantID 论文明确说"only use the largest face as reference facial landmarks"；PuLID、IP-Adapter FaceID 也是单脸设计。

2. **多脸方案的工程难度**：业界做法是 SAM2 / Grounded-SAM 把每个人分割出来 → 单独跑 ID 注入 → 拼回去。但：
   - 分割边界不精准 → 头发、耳朵处撕裂
   - 单独跑完再拼 → 光线/色调不一致
   - 身份互相"泄漏"（A 的 face embedding 信号通过 cross-attention 影响到 B 的区域）

3. **人脸检测在拥挤/遮挡场景失败**：如果一家人紧紧挤在一起、有人被遮挡了半边脸，人脸检测器可能漏检或错配。

### 法律风险等级

**低**。张冠李戴是体验 bug，不构成侵权。但如果换串的结果被用户发布且被误读为"他们两个人真的互换了身体"，存在极小概率的误导风险。

## 问题 4：非自愿换脸 / 深度伪造

### 用户视角（受害者）

"有人用我的照片做了换脸视频发在抖音上，我完全不知道。" → 名誉受损、心理伤害。

### 平台视角

**这是所有人脸 AI 特效面临的最核心安全问题。** 一旦发生高影响力深度伪造事件（如政治人物、明星色情视频），平台面临：
- 监管重罚（网信办约谈、下架整改）
- 法律诉讼（侵犯肖像权、名誉权）
- 品牌崩塌（用户信任归零）

### 技术根因

1. **没有活体检测或活体检测可绕过**：如果用户只需要上传一张静态照片就能做特效 → 随便从网上下载一张别人的照片就行。活体检测（要求前置摄像头实时拍）是第一道防线，但也不完美——高质量打印照片 / 屏幕翻拍在某些活体检测方案下可绕过。

2. **ID 注入本身不区分"是不是本人"**：InstantID / PuLID 的 ArcFace backbone 只提取"身份向量"，不判断"这张照片是不是你本人"——技术上无法从面部特征本身判断所有权。

3. **明星识别库覆盖不完整**：库里只有已知明星，网红、普通人不在库里 → 用普通人的脸做深度伪造，平台前置检测完全无法识别。

### 法律风险等级

**极高**。
- 《民法典》第 1019 条：未经本人同意不得制作、使用、公开他人肖像
- 《深度合成规定》第 14 条：深度合成服务提供者不得利用深度合成服务从事侵害他人合法权益的活动
- 司法实践中已有多起 AI 换脸判赔案例
- 刑事层面：涉嫌诽谤罪、侮辱罪（如果用于色情）

## 问题 5：NSFW 内容绕过过滤

### 用户视角

部分用户会**主动尝试**生成 NSFW 内容——通过精心构造的 prompt、特殊风格词、隐晦描述绕过文本过滤。

### 平台视角

一旦 NSFW 内容通过抖音特效生成 → 被保存 → 被发布 → 进入推荐流 → 其他用户（包括未成年人）看到 → 监管事故。

### 技术根因

1. **Prompt 层绕过**：用谐音、拆字、英文+中文混合、emoji 代替敏感词 → 文本过滤器识别不出意图。
2. **图像层对抗**：轻微改变生成参数（加特定 negative prompt、调 CFG、用特定 seed）可以让结果在 NSFW 边缘游走——通过后置过滤器阈值但人眼判断为擦边。
3. **后置过滤器的结构性缺陷**：
   - CompVis Safety Checker 基于 CLIP，仅覆盖 17 个"坏概念"，远不够中国合规要求
   - CLIP 系对"裸露皮肤面积"过度敏感 → 运动、泳装、健身内容误报率高
   - 对"暗示性但不直接裸露"的内容（擦边）识别率低
4. **开源模型的底层 NSFW 能力**：即使产品层没有 NSFW 数据微调，底层 backbone（SDXL、FLUX）从互联网数据训练继承了 NSFW 生成能力——彻底消除需要底层 unlearning，目前没有可靠方案。

### 法律风险等级

**高**。
- 涉未成年人的 NSFW 内容：**零容忍**，可触发刑事责任
- 一般 NSFW：违反《互联网信息服务管理办法》，平台承担主体责任
- 网信办通报 + 应用商店下架的先例已有多起

## 问题 6：名人 / 未成年人脸滥用

### 用户视角

- 粉丝用明星照片做"梦幻婚纱"特效 → 觉得好玩 → 不认为是侵权
- 家长用孩子照片做"长大后的样子"特效 → 结果被他人盗用

### 平台视角

**名人脸和未成年人脸是两条不可触碰的红线**：
- 名人：肖像权纠纷、deepfake 舆论风暴、经纪公司诉讼
- 未成年人：任何带有性暗示/成人化的生成结果 → 全球零容忍（中国《未成年人保护法》+ 各国 CSAM 法律）

### 技术根因

1. **明星识别库的覆盖问题**：
   - 中文明星 + 国际明星 + 政治人物 + 网红 → 库的量级巨大且需要持续更新
   - 昨天爆红的网红今天不在库里 → 时效性滞后
   - 化妆/P 图/老照片 → 识别率下降

2. **未成年年龄估计的误差**：
   - 公开的年龄估计模型（如 InsightFace 自带）误差 ±5 岁
   - 13-23 岁这个区间是最模糊的：18 岁的阈值两边各有 5 年误差带
   - 化妆、美颜滤镜让成年人显小 / 未成年人显大 → 阈值怎么设都有误判

3. **模型自己"脑补"名人脸**：即使用户没上传名人照片，模型训练数据里包含大量名人图 → 某些 prompt 下生成结果"碰巧"长得像某明星 → 需要后置二次检测。

4. **Lensa 教训（2022 年底）**：Lensa AI 上线时没做明星识别和未成年人保护 → 用户大量生成明星 NSFW 图 → 舆论崩塌 → 紧急下架整改。这是行业教科书级反面案例。

### 法律风险等级

**极高**。
- 名人：肖像权诉讼（民事）+ 名誉权诉讼 + deepfake 专项法规
- 未成年人：**全球最严标准**，涉 CSAM 可直接触发刑事责任，平台连带
- 中国特殊要求：《未成年人网络保护条例》明确禁止"利用网络侵犯未成年人肖像权"

## 问题 7：水印被剥离

### 用户视角

用户保存 AI 生成视频后：
- 截图 → 元数据丢失
- 录屏 → 显式水印被裁掉
- 转发到微信/微博 → 不同平台不互认 C2PA

### 平台视角

一旦"AI 生成"标识被去除 → 内容在互联网上作为"真实内容"传播 → 误导公众 → 监管追责回平台。

### 技术根因

1. **显式水印的脆弱性**：右下角"AI 生成"文字可被裁剪、遮挡、涂抹去除——任何图像编辑工具都能做到。

2. **C2PA 元数据的剥离问题**：
   - 社交平台传输时自动压缩 / 转码 → 元数据丢失
   - 用户手动"另存为"时很多格式不保留 manifest
   - 不同平台间没有统一的 C2PA 互认机制

3. **不可见水印（SynthID 类）的鲁棒性边界**：
   - 轻度修改（滤镜、轻微裁剪、压缩）→ 水印存活 ✅
   - 重度修改（大幅裁剪、截图再上传、AI 二次生成、强滤镜）→ 水印可能被破坏 ❌
   - 检测需要专用检测器 → 目前没有全行业统一的检测基础设施

4. **视频水印比图片更难**：视频经过平台转码、分辨率调整、帧率变化后，帧级别的不可见水印存活率更低。

### 法律风险等级

**中**。
- 平台已尽到标识义务（生成时确实加了水印）→ 用户主动去除后责任转移到用户
- 但监管可能追问"为什么水印这么容易被去除" → 技术升级压力
- 《深度合成规定》要求"不得删除、篡改、隐匿相关标识"——但执法能力有限

## 问题 8：合规风险（中国法规特殊要求）

### 用户视角

普通用户完全不了解合规要求，只觉得"我拍了一个好玩的特效视频就发了"。合规是平台的事。

### 平台视角

中国大陆的 AI 生成内容合规框架是**全球最严之一**，且处于快速演进中。核心法规：
- 《互联网信息服务深度合成管理规定》（2023.1.10 起施行）
- 《生成式人工智能服务管理暂行办法》（2023.8.15 起施行）
- 《个人信息保护法》（个人面部数据属于敏感个人信息）
- 《未成年人网络保护条例》（2024.1.1 起施行）

### 具体合规风险点

| 合规要求 | 风险描述 | 后果 |
|---|---|---|
| **算法备案** | 模型/算法上线前必须在网信办系统备案 | 未备案 → 下架 + 罚款 |
| **训练数据合法** | 训练数据来源必须合法；涉及个人信息须取得同意 | 数据来源有争议（如 LAION）→ 合规灰区 |
| **生成内容标识** | 必须在显著位置标识 + 元数据标识 | 未标识 → 网信办约谈 + 整改 |
| **不得生成违法内容** | NSFW、政治敏感、暴恐、谣言等 | 大量违规内容 → 下架 + 吊销资质 |
| **用户实名** | 提供深度合成服务须对用户实名验证 | 匿名用户滥用 → 平台连带责任 |
| **个人信息保护** | 用户面部数据属于"敏感个人信息"，收集需单独同意 | 违规收集/使用 → 高额罚款（营收 5%） |
| **未成年人保护** | 需要识别未成年人 + 限制服务范围 | 涉未成年人事故 → 社会舆论 + 监管重罚 |
| **跨境数据** | 面部数据原则上不得出境 | TikTok 和抖音共享模型/数据有风险 |

### 中国 vs 海外的关键差异

| 维度 | 中国大陆 | 欧美 |
|---|---|---|
| 监管节奏 | 快、先行先试、不断加码 | 慢、多以诉讼推动 |
| 标识要求 | **强制**、显著位置 + 元数据 | 自愿为主（EU AI Act 要求标识但尚未全面执行） |
| 算法备案 | **必须** | 无 |
| 人脸数据 | 敏感个人信息，单独同意 | GDPR 下属于生物特征数据，需明确合法基础 |
| 执法方式 | 行政约谈、下架、罚款、吊销 | 民事诉讼为主 |
| 政治敏感 | **独立维度**，其他国家无 | 不适用 |

### 法律风险等级

**极高**。合规是平台生存的前提条件，不是可选项。

## 问题之间的耦合关系

这些问题不是独立的，它们之间有张力：

1. **"像"vs"安全"**：身份保真度越高 → 换脸/伪造能力越强 → 安全风险越大。产品要的"60-70% 像"恰好是**安全和体验的平衡点**——像到足够有趣但不像到足以冒充。

2. **"误报"vs"漏报"**：NSFW/明星/未成年过滤器的阈值调严 → 误报率上升（正常用户被拦）→ 体验受损；阈值调松 → 漏报率上升 → 安全事故。**没有完美阈值，只有产品判断。**

3. **"速度"vs"安全"**：每多加一层检测（活体、明星识别、年龄估计、后置审核）→ 生成延时增加 → 用户等待体验变差。安全管线的延时预算是有限的。

4. **"覆盖"vs"成本"**：多人脸方案（分割 + 单独注入 + 拼接）→ 算力 3-5x → 算力成本是贯穿性约束。

---

## 产品设计师需要记住的

1. **安全不是"加个过滤器"的事**——是贯穿产品流程（上传 → 生成 → 保存 → 发布）每一步的设计决策。
2. **所有"体验问题"都有安全对偶**——不像是体验问题，太像是安全问题；速度快是体验追求，跳过检测是安全灾难。
3. **中国合规不是"和海外对齐"——是独立且更严的维度**。政治敏感、算法备案、强制标识、实名制——这些都是海外没有的。
4. **最大的风险不是技术漏洞，是用户意图**——大部分安全事故不是技术 bug，是用户有意绕过限制。产品设计要假设"有人会恶意使用"来设计防线。
