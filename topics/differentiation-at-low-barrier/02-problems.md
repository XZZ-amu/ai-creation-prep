# 低门槛下的差异化：常见问题

> 当产品试图同时做到"低门槛"和"有自己特色"时，会踩的坑。这些问题不是 bug，是结构性矛盾的表现。

---

## 1. 千人一面问题

### 现象

同一个热门特效上线后，前 1000 个用户的作品在 feed 流里几乎一模一样——同样的构图、同样的色调、同样的动效，只是脸不同。观众刷到第 5 个就开始划走。

### 为什么必然发生

- 低门槛 = 默认路径是固定的。用户走最省力的路径，而最省力的路径只有一条
- 热门特效的传播机制加剧同质化：看到别人用 → 我也用同样的 → 更多人看到 → 更多人用
- AI 模板本身的设计是"确定性的"——同一个模板的自由度被刻意压低以保证质量

### 真实案例

- **抖音 AI 写真（2023）**：上线初期爆火，但 3 天后 feed 流充斥同款"赛博朋克少女"，互动率断崖下跌
- **美图 AI 绘画**：不同用户的同一风格模板产出高度相似，社交平台上出现"撞脸"吐槽
- **Lensa AI Avatar**：全球火了一周后，Twitter 上开始出现"每个人的 AI 头像看起来都一样"的讨论

### 问题本质

**模板的"质量保证"和"差异化"是对冲的。** 模板越确定，质量越可控，但差异越小。开放度越高，差异越大，但质量越不可控。

### 衡量指标

- 同一特效的用户作品相似度（视觉 embedding 距离）
- 同一特效的互动率衰减曲线（第 1 天 vs 第 3 天 vs 第 7 天）
- 用户在 feed 流中对同特效内容的完播率变化

## 2. "选择困难"问题

### 现象

产品为了提供差异化，给了很多选项（10 个风格、5 个滤镜、3 个强度），结果用户在选项页停了 10 秒然后退出了。

### 为什么必然发生

- 普通用户缺少"审美词汇"——他们知道自己不想要什么（太丑的），但说不清想要什么
- 选项之间的差异如果不够大，用户分不出来；如果太大，用户怕选错
- "AI 生成"加剧了选择焦虑——因为不可撤销感更强（要再等 10 秒重新生成）

### 真实案例

- **即梦生图**：提供 20+ 风格模型选择，新用户的平均选择时间显著长于操作时间
- **Midjourney**：虽然是专业工具，但也因为"要写什么 prompt"的选择困难流失了大量轻度用户
- **美图写真**：早期把所有风格平铺展示，后来改成"推荐 3 个 + 查看更多"结构，转化率提升

### 问题本质

**差异化选项的价值 = 选项差异度 × 用户理解力 / 决策成本。** 当决策成本超过用户感知到的价值时，他们宁愿选默认或者不玩。

### 设计陷阱

| 做法 | 看起来 | 实际效果 |
|------|--------|---------|
| 平铺 20 个风格 | "给用户自由度" | 选择瘫痪，流失率上升 |
| 用术语命名风格（"赛博朋克""蒸汽波"） | "专业感" | 不懂的用户直接跳过 |
| 每个选项都展示预览图 | "降低选择成本" | 预览图加载时间 + 视觉噪音 |
| 只给 2 个选项（A/B） | "简化选择" | 差异化空间过小，回到千人一面 |

### 衡量指标

- 选项页停留时间 vs 完成率的关系
- "未选择直接退出"的比例
- 默认选项的使用率（如果 >80%，说明其他选项是噪音）

## 3. 差异化但丑

### 现象

用户利用产品提供的自由度做出了"确实不一样"的东西，但审美质量很差——变形的脸、不协调的色彩、诡异的构图。用户发了，没人互动，下次不来了。

### 为什么必然发生

- 审美不是均匀分布的——大部分参数组合的结果都是丑的，好看的只占一小片区域
- 用户没有"预判能力"——调参数时不知道结果会怎样，调完等 10 秒看到丑图才后悔
- AI 模型本身对"超出训练分布"的输入容易出瑕疵（artifacts）

### 真实案例

- **Stable Diffusion WebUI 的 CFG Scale**：稍微调高就出色彩过饱和、面部变形，但用户不知道边界在哪
- **抖音某些美颜特效**：用户把"大眼"拉到最大 → 恐怖谷效应
- **可灵的自由图生图**：给太多自由度后，用户上传质量差的参考图 + 模糊 prompt = 输出不可控

### 问题本质

**"审美安全区"远比"技术可行区"小。** 模型能生成的范围远大于好看的范围。产品设计的工作是把用户限制在安全区内，同时让他们感觉自己有选择。

### 与千人一面的矛盾

这个问题和问题 1 形成双杀：
- 限制自由度 → 千人一面
- 放开自由度 → 差异化但丑
- 产品需要找到"自由度足够差异化、但怎么调都不太丑"的设计空间

### 衡量指标

- 用户自定义作品 vs 默认作品的互动率对比
- "生成后未发布"的比例（可能是生成了但觉得丑）
- 用户调参后的重新生成次数（高 = 不满意当前结果）

## 4. 新鲜感衰减

### 现象

一个特效第一天在 feed 流里是"wow 这什么"，第三天是"又是这个"，第七天是"能不能别再发了"。差异化在时间维度上失效。

### 为什么必然发生

- 人类对视觉刺激的适应极快（心理学：habituation）
- 社交媒体的传播机制加速了曝光饱和——一个火的特效 24 小时内可以被百万人看到
- AI 特效的"产出一致性"比传统滤镜更高（模板结构更强），所以看起来更"同款"

### 真实案例

- **"AI 变老"特效（FaceApp）**：2019 年全球病毒传播，持续热度约 5 天后彻底消退
- **Lensa AI Avatar**：2022 年底全球爆火约 10 天，此后用户留存暴跌
- **抖音各类 AI 写真模板**：通常热度周期 3-5 天，需要持续上新

### 生命周期模型

```
热度
 │
 │    ╱╲
 │   ╱  ╲
 │  ╱    ╲
 │ ╱      ╲────────────────
 │╱
 └──────────────────────────── 时间
   D1  D3  D5  D7+
   爆发  衰减  稳态/消亡
```

### 问题本质

**低门槛加速了"消费"速度。** 门槛越低 → 更多人同时用 → 内容同质化越快 → 新鲜感消退越快。这是低门槛策略的结构性副作用。

### 对供给策略的要求

- 不能靠"一个爆款特效吃一个月"——需要高频上新
- 每个特效的设计都要考虑"第 N 天看到第 M 个人用还有没有意思"
- 需要在特效本身的机制里埋入时间维度的差异化（今天用和明天用结果不同？）

### 衡量指标

- 特效热度半衰期（从峰值掉到 50% 的天数）
- N 天后用户主动使用率（而非被推荐后使用）
- 同一特效的互动率逐日变化曲线

## 5. 素材质量限制

### 现象

产品设计了一个很好的"素材驱动"方案——用户上传自己的照片，AI 把它变成某种风格。但很多用户的原始素材质量太差：模糊、光线暗、角度奇怪、背景杂乱。AI 也救不了。

### 为什么必然发生

- 抖音的普通用户不是摄影师——手机前置摄像头、室内灯光、随手一拍
- "素材驱动差异化"的前提是素材本身有足够的信息量。低质量素材 = 低信息量 = AI 只能"脑补"
- AI 的"脑补"要么太保守（模糊的地方直接糊掉）要么太激进（脑补出不属于用户的细节）

### 真实案例

- **AI 写真类产品**：侧脸、遮挡、低光照的自拍 → 生成结果面部变形或不像本人
- **AI 宠物类特效**：用户的宠物照片背景太杂 → AI 分不清主体
- **AI 风格化视频**：原始视频画质低（压缩严重）→ 风格化后噪点被放大

### 矛盾结构

```
用户素材质量分布：

数量
 │  ████
 │  ██████
 │  ████████
 │  ██████████
 │  ████████████
 │  ██████████████
 └──────────────────────── 质量
    差  中  好  优
    
    ← 大多数用户在这里
```

产品面对的不是"平均素材"，是"最差的那 50% 素材"。如果产品只在好素材上表现好，就只能服务一半用户。

### 问题本质

**素材驱动差异化的上限取决于素材质量分布的下限。** 你不能假设用户会提供好素材——他们不会。

### 应对方向（待在 03-solutions 中展开）

- 前置引导：在用户拍摄/上传前告诉他"什么样的素材效果好"
- 智能兜底：对低质量素材降级到更宽容的处理方式（比如只用大轮廓、不用细节）
- 素材增强：先用超分/修复模型把素材质量拉上来，再做风格化
- 换一个不依赖素材质量的差异化来源

### 衡量指标

- 不同素材质量等级的生成成功率
- "上传后放弃"比例（可能是看到预览效果差）
- 用户二次拍摄/更换素材的比例

## 问题之间的关系

这 5 个问题不是独立的，它们形成了一组互锁的约束：

```
千人一面 ←──── 放开自由度 ────→ 差异化但丑
    │                                   │
    │  ← 加选项对冲 →                    │
    │                                   │
    ▼                                   ▼
选择困难                              重试/放弃
    │                                   │
    └───── 体验差 → 用户不来了 ──────────┘
    
新鲜感衰减 = 时间维度的千人一面
素材质量限制 = 差异化来源本身的瓶颈
```

**核心 takeaway**：不能只解一个问题——解千人一面可能引发选择困难，解选择困难可能回到千人一面。需要的是在这些约束之间找到"可居住的设计空间"。
