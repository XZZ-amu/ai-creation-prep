# 生成质量与兜底：常见问题与失败模式

> AI 生成不是"出来就完美"，而是"每一次都在和各种崩坏概率对赌"。产品设计者的工作是理解这些失败的规律性，才能设计出有效的防线。

---

## 一句话总结

**AI 生成的失败不是随机的——每类问题都有明确的技术根因和出现规律，理解它们的频率和严重程度才能决定产品层面"兜底的优先级"。**

---

## 问题总览

| 问题类别 | 用户感知严重度 | 出现频率（2024 主流模型） | 新模型缓解程度 |
|---|---|---|---|
| 手部/手指畸变 | ⭐⭐⭐⭐⭐ | 中（FLUX/SD3 后降低） | 明显缓解但未根除 |
| 人脸崩塌/不对称 | ⭐⭐⭐⭐⭐ | 低-中 | 大幅缓解 |
| 文字渲染乱码 | ⭐⭐⭐⭐ | 高（FLUX 前）→ 中低（FLUX 后） | 明显缓解 |
| 物理不合理 | ⭐⭐⭐ | 中 | 部分缓解 |
| 细节模糊/过度平滑 | ⭐⭐⭐ | 中 | 部分缓解 |
| 风格不稳定 | ⭐⭐⭐⭐ | 高 | 未显著改善 |
| 视频特有问题 | ⭐⭐⭐⭐⭐ | 高 | 改善中但仍突出 |

## 1. 手部/手指畸变

### 用户感知

"六根手指"、"手指像面条一样粘在一起"、"拿东西手指穿进物体里"。这是公众对 AI 生成质量最直觉的吐槽点——"手能看出是 AI 的"已经成为一种文化 meme。

### 技术原因

- **训练数据中手部姿态极其多样**：不同于人脸（相对标准化的结构），手有 21 个关节点、无限种握姿、互相遮挡的可能性——模型无法从有限训练图中学到"永远是 5 根手指"的硬约束
- **Diffusion 模型的生成方式没有"计数能力"**：去噪过程是连续的像素预测，不是先数"几根手指"再画。模型不知道"已经画了 5 根"——它只知道"这个像素区域大概长什么样"
- **手指互相遮挡导致歧义**：当 3 根手指叠在一起时，模型不确定边界在哪里，容易"多生成"或"少生成"

### 出现频率

- SD 1.5 时代：**几乎必出**，大约 40-60% 的含手生成有明显问题
- SDXL 时代：**中频**，约 20-30%
- FLUX / SD3 时代：**低-中频**，约 10-15%，简单姿势已基本正常，复杂姿势（弹琴、握拳叠加、物件交互）仍频繁出错
- 视频场景：频率比静图更高，因为动态中手部姿态变化增加了出错概率

### 严重程度：极高

用户对手指错误的容忍度极低——这是"一眼假"的标志。在抖音特效场景中，如果用户看到自己的变身图/视频手指是歪的，**发布意愿直接归零**。

### 新模型缓解情况

FLUX 和 SD3 通过更大的模型容量（DiT 架构）和更高质量的训练数据显著改善了手部质量。但物理约束（5 指结构）仍非内建——模型本质上还是在"统计猜测"而非"结构推理"。来源：多方用户测评 + Stability AI 公开 demo 对比（2024）。

## 2. 人脸崩塌/不对称

### 用户感知

"脸完全变形了"、"两边眼睛不一样大"、"嘴巴位置不对"、"变成了恐怖谷"。对抖音 AI 特效而言尤其致命——用户上传的是自己的脸，出来变成"怪物脸"是最差体验。

### 技术原因

- **面部对称性是极精细的像素级约束**：眼距差 2 个像素人就能感知不对，但模型在 latent space 操作，decode 回像素时的误差可能恰好打在面部关键位置
- **多张脸场景更容易崩**：当画面中有多人时，模型的 attention 分散，远处的脸或小脸更容易变形
- **风格化和真实之间的冲突**：特效要求"变化"（魔法少女化），但变化太多就丢了面部结构；变化太少又没特效效果——这个张力导致人脸区域成为"风格化 vs 结构保持"的战场
- **Identity 注入（IP-Adapter Face、InstantID）的保真度有上限**：这些模型只能把面部特征作为"参考"注入生成，不是像素级复制

### 出现频率

- 正脸、单人、标准光照：**低频**（< 5%），现代模型对标准人脸已经很稳定
- 侧脸、遮挡、表情夸张：**中频**（15-25%）
- 多人场景中的非主体脸：**高频**（30-50%）
- 小尺寸脸（远景或画面边缘）：**高频**（40%+）

### 严重程度：极高

人类视觉系统对面部异常极其敏感——进化决定的。0.5 秒就能判断"这个脸不对"。在抖音特效中，用户上传自拍的核心期望就是"像我 + 好看"，任何面部崩塌都是致命打击。

### 新模型缓解情况

FLUX / SD3 对单人正脸的生成质量已经非常稳定。但一旦涉及"保持用户 identity + 风格变化"的组合需求（恰恰是抖音特效的核心需求），问题依然存在。InstantID / PhotoMaker 等方案在持续改进中，但"像不像用户"仍是未完全解决的问题。

## 3. 文字渲染乱码

### 用户感知

"生成的图里有文字，但是乱码"、"字形对了但顺序全乱"、"中文直接变成符号"。当特效需要生成包含文字的场景时（比如"海报风格"、"杂志封面"），这个问题非常突出。

### 技术原因

- **CLIP text encoder 不理解字符顺序**：CLIP 把文字理解为语义向量，不是字符序列。"HELLO"和"OLLEH"在语义空间的距离可能很小
- **Diffusion 模型没有"排版能力"**：它只知道"这个区域大概有字形"，但不知道具体是哪个字、什么顺序
- **中文比英文更难**：中文字符结构复杂度远高于拉丁字母，训练数据中包含正确中文文本的图比例较少
- **字体多样性问题**：模型见过无数种字体风格，生成时可能在不同字之间切换字体

### 出现频率

- SD 1.5 / SDXL：**极高频**，几乎无法正确渲染任何文字
- FLUX：**明显改善**，简单英文单词可以正确生成，但中文和长句仍频繁出错
- DALL-E 3：**大幅改善**，英文文字准确率较高，中文仍不稳定
- Ideogram：**专项优化**，文字准确率目前最高（来源：社区评测对比，2024）

### 严重程度：高

取决于场景。如果特效设计不涉及文字元素，这个问题不存在。但如果特效是"生成同人志封面"、"明星应援海报"等文字密集场景，乱码直接让产出不可用。

### 新模型缓解情况

这是近一年进步最大的领域之一。FLUX 使用 T5-XXL text encoder（真正理解文字序列），文字渲染能力跨代提升。DALL-E 3 通过额外的文字渲染模块也大幅改善。但中文渲染、复杂排版仍是薄弱环节。

## 4. 物理不合理（浮空、穿模、透视错误）

### 用户感知

"人飘在空中没有影子"、"手穿过桌子"、"两个物体的前后关系不对"、"远处的东西比近处的还大"、"头发穿过肩膀"。

### 技术原因

- **模型从 2D 图像学习，不理解 3D 物理**：Diffusion 模型学的是"图片像素长什么样"，不是"物体在 3D 空间怎么摆放"。它见过足够多"人站在地上"的图，所以大多数时候站对了，但不是因为它理解重力
- **遮挡关系需要 3D 推理**：A 在 B 前面所以挡住 B 的一部分——这需要空间理解，模型只是在统计"这两个物体同时出现时通常谁挡谁"
- **复杂场景的组合爆炸**：训练数据里"人骑马"的图很多，但"人骑着马同时举着伞且马旁边有条狗"的图很少——组合越复杂，模型靠统计记忆的能力越不够

### 出现频率

- 简单场景（单人、常见姿势）：**低频**（< 5%）
- 多物体交互场景：**中频**（15-30%）
- 非常规物理场景（倒挂、水下、失重）：**高频**（40%+）
- 抖音特效常见场景（单人 + 背景替换）：**低-中频**，大部分时候不严重

### 严重程度：中

用户对物理错误的容忍度比对面部错误高——尤其在"特效"语境下，用户本来就预期"不真实"。一个魔法少女浮空可能反而是"对的"。但明显的穿模（手穿过物体）仍会被吐槽。

### 新模型缓解情况

视频生成模型（Sora、可灵）通过大规模视频训练开始建立初步的物理直觉（因为视频数据天然包含物理运动规律）。但 3D 理解仍然是统计层面的，不是真正的物理引擎。待验证：字节内部视频模型在物理一致性上的具体表现。

## 5. 细节模糊/过度平滑

### 用户感知

"看着像被美颜磨皮过了"、"远处的细节全是糊的"、"头发丝糊成一坨"、"衣服纹理像画出来的没有质感"。整体能看，但放大或仔细看就"假"。

### 技术原因

- **VAE 解码器的信息瓶颈**：Latent Diffusion 在压缩空间工作（通常 8× 下采样），VAE decoder 还原回像素时必然有信息损失。高频细节（头发丝、皮肤纹理、布料纤维）是最先被牺牲的
- **去噪步数不足**：步数少则每一步跳跃大，细节来不及"画清楚"就已经结束了。快速生成（LCM、Turbo、少步数调度器）的代价就是细节损失
- **超分模型的过度平滑**：Real-ESRGAN 等超分网络有"去噪过强"的倾向——它把高频噪点和高频细节一起抹掉了，出来的图干净但"塑料感"
- **RLHF/DPO 副作用**：经过人类偏好对齐的模型倾向于生成"安全的"——对比度适中、饱和度适中、整体平滑——的图。这种"统计上讨好评委"的倾向会牺牲细节的丰富度

### 出现频率

- **中频**：几乎所有快速生成模式（步数 < 20）都有一定程度的细节损失
- 在抖音特效"1-2 秒出图"的场景下，由于步数必须压缩，这个问题是系统性的
- 视频更严重：为了控制帧间一致性，时序平滑滤波会进一步"磨平"细节

### 严重程度：中

用户在手机屏幕上看（1080×1920）通常不会太在意——手机屏幕足够小，细节损失不如在电脑大屏上明显。但如果用户截图放大分享，问题会暴露。对于"高质量特效"定位的场景，这是一个隐性不满——用户说不出"哪里不对"，只觉得"假"。

### 新模型缓解情况

FLUX / SD3 通过更大的 latent 空间和更强的 VAE 改善了细节表现。但在快速生成场景下（步数 < 10），细节损失仍然明显。这是"速度 vs 质量"的根本矛盾，不是模型能力问题。

## 6. 风格不稳定（同 prompt 质量波动大）

### 用户感知

"同样的设置，出来 4 张完全不一样"、"上一次出来好看这次出来丑"、"用同一个特效，我的图好看朋友的图丑"。用户期望的是"稳定的好"，但得到的是"开盲盒"。

### 技术原因

- **随机种子（seed）决定初始噪声**：每次生成从不同的随机噪声出发，最终结果天然不同。这是 diffusion 模型的设计——多样性就是靠 seed 变化来的
- **Prompt 的语义空间存在"宽度"**：同一句文字描述对应 latent space 里的一个区域而非一个点——这个区域内的不同位置会生成不同的图
- **模型在 prompt 边界情况下表现不稳定**：当 prompt 描述的场景位于训练数据的"稀疏区"（不常见的组合），模型的输出方差剧增
- **CFG scale（引导强度）设置的影响**：CFG 太低则 prompt 控制力弱（随机性大），CFG 太高则过度饱和且画面僵硬

### 出现频率

- **高频**：这本质上不是"bug"，而是 diffusion 模型的特性。任何没有固定 seed 的生成都有波动
- 波动幅度取决于 prompt 的具体性：描述越精确、参考图越强，波动越小
- 抖音特效里如果 LoRA + 参考图控制得当，波动可以被压到用户可接受范围

### 严重程度：高

这是影响用户信任的核心问题。用户看到别人用同一个特效出来好看的图，自己试了出来丑的，**第一反应不是"随机的"，是"AI 对我不好"**。这种不公平感极其损害产品口碑。

### 新模型缓解情况

**没有显著改善，因为这不是 bug 而是 feature。** 缓解手段在产品层而非模型层：Best-of-N 选图、更强的 conditioning（ControlNet/参考图）、固定 seed 范围等。RLHF/DPO 通过提升下限（让最差的也不太差）间接缓解了部分问题。

## 7. 视频特有问题

视频生成比图片多了"时间维度"，带来一整类图片生成不存在的问题。

### 7a. 闪烁（Flickering）

**用户感知**："视频忽明忽暗"、"颜色一跳一跳的"、"感觉在不停闪"。

**技术原因**：逐帧独立生成时，每帧的颜色/亮度基于各自的噪声出发点，没有全局约束保证帧间一致。即使用了时序约束（temporal attention），在光照变化或运动剧烈的区域，模型仍难以保持稳定。

**出现频率**：中-高频。静态场景少，动态场景多。几乎所有当前视频生成模型（Runway Gen-3、可灵、即梦视频）在运动剧烈片段都有不同程度闪烁。

### 7b. 主体抖动/变形（Subject Jitter）

**用户感知**："人脸在抖"、"身体轮廓一帧一帧在变"、"整个人在果冻一样晃动"。

**技术原因**：时间维度上的一致性需要模型"记住"前几帧的主体形状——当前的 temporal attention 机制有限的感受野难以做到完美。尤其是主体转身、被遮挡再出现时，模型不确定"应该长什么样"，导致形状在帧间微小变化。

**出现频率**：中频。短视频（< 3 秒）相对可控，长视频（> 5 秒）几乎必出。

### 7c. 物体凭空消失/出现

**用户感知**："背景里的椅子突然没了"、"手里的东西中途消失了"、"画面里多出一个不存在的物体"。

**技术原因**：视频模型的 temporal attention 对"非主体物体"的跟踪能力有限。注意力主要集中在主体（人物、前景），背景物体和小道具容易在帧间"被遗忘"。

**出现频率**：中频。在 3-5 秒以上的生成中尤其明显。可灵、Runway Gen-3 的用户社区中这类报告频繁出现。

### 7d. 运动不自然

**用户感知**："人走路像在滑行"、"动作太慢像慢动作"、"手臂动了但身体没跟着动"。

**技术原因**：视频模型学习的是"什么运动看起来像真的"，但真实运动是由骨骼、肌肉、惯性等物理规律驱动的。模型没有人体运动学知识，只是在模仿训练数据中的运动模式。复杂运动（舞蹈、体育动作）在训练数据中稀疏，生成质量差。

**出现频率**：中-高频，取决于运动复杂度。简单转头、微笑等表情变化比较稳定；全身运动、复杂手势高频出错。

### 视频问题的综合严重程度：极高

视频问题比图片问题更致命，因为**视频是时间连续的——一个 0.5 秒的崩坏会被用户明确注意到，无法像图片一样"整体还行就接受"**。用户对视频的质量期望锚定在日常看到的短视频上——对标的是真实拍摄的内容。

### 新模型缓解情况

2024-2025 是视频生成质量快速迭代的时期：
- **可灵 1.5 / 1.6**：主体一致性和运动自然度相比初版大幅改善（来源：快手官方发布对比）
- **Runway Gen-3 Alpha Turbo**：闪烁问题明显改善，但运动复杂场景仍有抖动
- **Sora**（OpenAI）：从 demo 看物理一致性领先，但实际开放后用户反馈仍有各种问题（来源：社交媒体用户反馈，2025）
- **整体趋势**：每 3-6 个月一次代际改善，但"完全无瑕疵视频"仍未实现

## 问题之间的关联性

这些问题不是孤立的，它们之间有系统性的联动：

### 速度 vs 质量的根本矛盾

| 追求 | 牺牲 |
|---|---|
| 更快出图（少步数） | 细节模糊 ↑、手部错误 ↑ |
| 更多候选（Best-of-N） | 等待时间 ↑、算力成本 ↑ |
| 更强风格化 | 人脸变形风险 ↑、物理合理性 ↓ |
| 更高一致性（强约束） | 多样性 ↓、创意空间 ↓ |

### 修复可能引入新问题

- 修脸 → 可能改变 identity（"修成另一个人"）
- 超分 → 可能过度平滑（"塑料感"）
- 帧间平滑 → 可能丢失运动细节（"果冻感"）
- 合规过滤 → 可能误杀正常内容（"明明没问题被拒了"）

---

## 对抖音特效场景的优先级排序

基于"用户上传自拍 + 选特效 + 一次出结果"的核心场景，问题严重度排序：

1. **人脸崩塌** — 致命，用户核心期望是"像我 + 好看"
2. **视频闪烁/抖动** — 致命，视频场景下这是最常被吐槽的
3. **手部畸变** — 严重，"一眼假"的标志
4. **风格不稳定** — 严重，损害产品信任感
5. **细节模糊** — 中等，手机屏幕可容忍，但影响"高级感"
6. **物理不合理** — 中等，特效语境下容忍度较高
7. **文字渲染** — 看场景，多数特效不涉及文字

**关键洞察**：前 4 项需要**产品层面主动兜底**（不能指望模型完美），后 3 项可以靠模型迭代逐步改善。
