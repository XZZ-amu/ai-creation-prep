# 迭代精准性：常见问题与失败模式

> 用户对已生成内容做局部修改时，哪些环节会翻车？

---

## 一句话总结

**迭代修改的核心矛盾是"改动的自由度"：改得太自由 → 牵连无辜区域；锁得太死 → 新意图进不来。所有失败模式本质上都是这条光谱上的失衡。**

---

## 问题分类总览

| 编号 | 问题 | 一句话 | 严重程度 |
|------|------|--------|----------|
| P1 | 联动漂移 | 改了 A，B 也变了 | ★★★★★ |
| P2 | 欠编辑 | 改了但没改到位 | ★★★ |
| P3 | 过编辑 | 改过头了 | ★★★★ |
| P4 | 多轮累积漂移 | 编辑几轮后整体走样 | ★★★★★ |
| P5 | Mask 边界不自然 | 拼接痕迹明显 | ★★★ |
| P6 | 视频帧间不一致 | 改完闪烁或跳变 | ★★★★ |

## P1：联动漂移（改了 A，B 也变了）

### 现象

用户说"把围巾改成蓝色"，围巾确实变蓝了，但背景的咖啡馆变成了另一家、猫的姿势微妙变化、窗外的雪粒位置全换了。**用户的感受是"你动了我没让你动的东西"。**

### 原因

1. **cross-attention 的全局性**：在 diffusion model 里，改一个 token 会导致 cross-attention map 在所有空间位置重新打分。即使只改了"red→blue"，其他位置的 attention 分布也会变——导致去噪路径整体偏移。这是 Prompt-to-Prompt 论文（Hertz et al., 2022）反复展示的核心问题。

2. **端到端重生成的本质**：InstructPix2Pix、img2img、FLUX Kontext 等方法都是"输入原图 + 条件 → 生成一张新图"。新图里的每个像素都是重新采样的，不存在"物理性锁定"——只是通过训练让模型学会"参考原图"，但"参考"不等于"复制"。

3. **Latent space 的纠缠**：VAE 把图片压缩到 latent space 时，不同语义区域的信息是纠缠的。改变一个区域的 latent 值，解码回像素时可能影响相邻区域。

### 触发条件

- 使用 img2img（strength > 0.3）时几乎必然发生
- InstructPix2Pix 在颜色/风格类编辑时轻微、结构类编辑时严重
- Prompt-to-Prompt 在 τ 设置不当时发生
- FLUX Kontext 在简单编辑时控制较好，复杂编辑时仍会出现

### 严重程度：★★★★★

这是用户放弃迭代编辑的首要原因。**用户心智模型是"PS 式的图层编辑"——改哪算哪，不该动的绝对不动。但 AI 生成模型的心智模型是"重新画一张参考原图的新图"。** 这个预期差距是产品体验的核心痛点。

### DiT 时代是否缓解

**部分缓解。** FLUX Kontext 通过 token concat + 大规模编辑对数据训练，让模型更好地学会"哪些该保留"。论文报告 character consistency 和 background preservation 显著优于 InstructPix2Pix。但 Kontext 仍然是端到端重生成——"参考"质量提高了，但不等于"锁定"。在像素级保真要求下仍然不够。

## P2：欠编辑（改了但没改到位）

### 现象

用户说"把围巾改成蓝色"，结果围巾变成了蓝灰色、或者只有一半变蓝、或者看起来像"蒙了一层蓝色滤镜"而不是真的换了材质。用户感受：**"你听见了但没做好"。**

### 原因

1. **img2img strength 太低**：strength < 0.2 时，从原图加的噪声太少，新 prompt 几乎没有空间发挥——去噪过程直接回到接近原图的状态。

2. **Guidance scale 冲突**：InstructPix2Pix 有两个 guidance——image guidance（$s_I$）让输出像原图，text guidance（$s_T$）让输出遵从指令。$s_I$ 太大会压制 $s_T$，导致编辑力度不够。

3. **Attention 注入太强**：Prompt-to-Prompt 中 τ 设置太大（>0.8），原图的 attention map 把新词的影响几乎完全压制。

4. **训练数据偏差**：InstructPix2Pix 的训练数据是合成的，很多编辑对的变化幅度本身就小——模型学到了"保守编辑"的 prior。

5. **指令理解不足**：用户说"换成赛博朋克风格的围巾"，模型不理解"赛博朋克"具体要什么变化，只做了颜色微调。

### 触发条件

- img2img strength < 0.25
- InstructPix2Pix 的 $s_I$ > 1.5 且 $s_T$ < 7
- 编辑指令涉及抽象概念（"更有质感"、"更高级"、"赛博朋克"）
- 编辑区域很小（模型倾向于忽略小变化）

### 严重程度：★★★

比联动漂移轻——至少没搞坏其他部分。但多次欠编辑会消耗用户耐心，形成"这玩意儿不听话"的印象。在抖音场景下，用户会直接放弃而不是反复尝试。

### DiT 时代是否缓解

**有改善。** FLUX Kontext 和 Step1X-Edit 都用了更强的文本编码器（T5-XXL / Qwen-VL），对编辑意图的理解能力更强。Step1X-Edit 用 MLLM 解析指令，能把模糊指令转化为更明确的编辑信号。但对于真正抽象的指令（"让它更有氛围感"），仍然会欠编辑。

## P3：过编辑（改过头了）

### 现象

用户说"让围巾颜色深一点"，结果围巾变成了完全不同的黑色皮革质感；或者说"加一点雪花"，结果变成暴风雪。用户感受：**"我只是想微调，你给我大改了"。**

### 原因

1. **Guidance scale 过高**：text guidance 或 classifier-free guidance 过高时，模型过度响应指令，把小调变成大改。

2. **img2img strength 过高**：strength > 0.6 时，原图信息被噪声淹没太多，去噪过程等于半重画。

3. **训练数据的分布不均**：InstructPix2Pix 训练数据里"大改"比"微调"多（合成数据更容易生成大变化对），导致模型 prior 偏向激进编辑。

4. **DiT 模型的"过度理解"**：Step1X-Edit 用 MLLM 解析指令时，有时会把简单指令过度展开——"加一点雪花"被理解为"场景应该是下雪天"，导致全局改变。

### 触发条件

- img2img strength > 0.5
- InstructPix2Pix 的 $s_T$ > 12
- 指令中包含程度词但模型无法精确理解程度（"一点"、"稍微"、"略微"）
- 编辑涉及风格/氛围类词汇（模型倾向于大幅改变以确保"看得出差别"）

### 严重程度：★★★★

比欠编辑更严重——过编辑往往不可逆，用户必须从头来过。在多轮迭代场景下尤其致命：第三轮过编辑意味着前两轮的工作全白费。

### DiT 时代是否缓解

**改善有限。** FLUX Kontext 在保留度上做得好（不容易把不该改的改了），但在"编辑力度控制"上没有专门优化。缺乏一个用户可调的"编辑强度"旋钮是 DiT 时代编辑模型的共性短板——Step1X-Edit 论文里也没有提供精细的强度控制接口。

## P4：多轮累积漂移（编辑几轮后整体走样）

### 现象

第一轮改围巾颜色 → 第二轮加帽子 → 第三轮调背景 → 第四轮发现猫已经不是原来那只猫了——脸型变了、毛色偏了、整体构图都漂移了。用户感受：**"前面改得好好的，改着改着就崩了"。**

### 原因

1. **误差累积的数学必然**：每一轮编辑的输出都会成为下一轮的输入。每一轮都有微小偏差（latent 空间的非精确映射），N 轮后偏差累积到肉眼可见。这不是某个模型的 bug，是所有"输出当输入"迭代系统的固有特性。

2. **无锚点漂移**：端到端重生成类方法（InstructPix2Pix、Kontext）没有"原始锚点"概念——每轮只看上一轮的输出，不看最初始的图。丢失了"出发点"。

3. **VAE 编解码损失**：每一轮编辑都经历 encode → latent 操作 → decode。VAE 的重建不是无损的，多次编解码会累积失真（类似多次 JPEG 压缩）。

4. **Identity 信息没有显式锚定**：diffusion model 不理解"这是同一只猫"——它只是在 latent space 里做去噪。多轮后，"猫"的 identity 自然漂移。

### 触发条件

- 编辑轮次 > 3 时普遍出现
- 每轮编辑的 strength / guidance 越高，漂移越快
- 涉及面部/角色的编辑特别明显（人对面部变化极敏感）
- 非 inpainting 方法（没有像素级锁定）更严重

### 严重程度：★★★★★

**这是"迭代精准性"这个 topic 的核心难题。** 用户的真实使用路径是多轮微调直到满意——如果 3 轮后就崩盘，产品的"迭代修改"功能形同虚设。FLUX Kontext 论文专门把"multi-turn editing with minimal drift"作为卖点，反过来证明行业认为这是未解决问题。

### DiT 时代是否缓解

**显著缓解但未消除。** FLUX Kontext 在 KontextBench 上展示了多轮编辑后 character consistency 和 background preservation 明显优于 GPT-Image-1 和 Gemini Native Image Gen。Step1X-Edit 也报告了类似的改善。但论文自己也说是"minimal drift"——不是零漂移。5 轮以上的迭代仍然会出问题。

**缓解的核心原因**：DiT 的 token concat 架构让模型在生成时能直接"看到"原图的完整 latent 表征（通过 self-attention），比 UNet 时代靠 cross-attention 或额外通道传递原图信息强得多——信息损失更少，保留度更高。

## P5：Mask 边界不自然（拼接痕迹明显）

### 现象

用户画了 mask 做 inpainting，改完后 mask 边界出现明显的色差、光线不连续、纹理断裂——看起来像 PS 时手动贴了一块上去。用户感受：**"假，一看就是 P 的"。**

### 原因

1. **非 inpainting 专用模型的硬贴机制**：标准的 inpainting 后处理是把 mask 外的原始像素直接贴回去。mask 内是重新生成的，颜色/光线/纹理和 mask 外不一定匹配——硬贴就有接缝。

2. **Mask 形状和语义边界不对齐**：用户在手机上画 mask 很难精确——画到了猫毛和围巾的交界处，一部分猫毛被重画，和原来的猫毛颜色/方向不一致。

3. **feathering 和 blending 参数不当**：Inpainting pipeline 通常有 mask blur / feather 参数来软化边界。参数太小 → 硬边；参数太大 → mask 区域扩大，改了不该改的。

4. **生成内容和周围光照不一致**：mask 内重画时，模型可能没正确理解周围的光源方向。围巾上的高光方向和猫毛的高光方向不一致 → 露馅。

### 触发条件

- 使用非 inpainting 专用模型 + overlay 硬贴
- Mask 边界穿过复杂纹理区域（毛发、布料褶皱）
- 手机端 mask 工具精度不够（手指涂抹 vs 画笔精确描边）
- 被替换内容和原内容的光照/透视差异大

### 严重程度：★★★

用户对边界自然度的敏感程度因内容而异——纯色区域边界不太明显，复杂纹理区域非常明显。在抖音场景下，视频压缩（H.264/H.265）会掩盖一部分接缝，但高清大图上很致命。

### DiT 时代是否缓解

**明显缓解。** FLUX.1 Fill 是 DiT 时代的 inpainting 专用模型，在训练时就把 mask + 原图作为输入条件，模型从一开始就学会"接边界"。相比 UNet 时代的"先生成再硬贴"，DiT inpainting 模型的过渡自然度大幅提升。但仍然存在：当 mask 边界穿过高频纹理（毛发）时，自然过渡仍然是难题。

## P6：视频编辑的帧间不一致（改完闪烁或跳变）

### 现象

对视频做局部修改后，被改的区域在不同帧之间颜色跳变、形状闪烁、或者某几帧突然回到原样。用户感受：**"改是改了，但一播放就露馅"。**

### 原因

1. **逐帧独立生成的采样不一致**：如果对每帧独立做 inpainting，每帧的 mask 区域是独立采样的随机过程——哪怕条件相同，输出也会帧间不一致。这是闪烁的根本来源。

2. **Mask 追踪丢失**：视频中物体在运动。第一帧画的 mask 需要追踪到后续帧。当目标快速移动、被遮挡、或形态变化时，追踪器（SAM-2、视频分割模型）丢点 → mask 跳到错误位置 → 那几帧的编辑结果完全错误。

3. **时间维度的去噪不一致**：Video diffusion model 理论上通过 temporal attention 保证帧间一致，但实际效果受限于 temporal window 大小。超出 window 的帧之间仍可能不一致。

4. **关键帧编辑 + 重生的传播误差**：修改首帧后让 video 模型重生整段视频，模型可能"渐忘"首帧的编辑——越往后的帧越偏向原始内容。

### 触发条件

- 目标在视频中快速运动（mask 追踪容易掉）
- 视频长度 > 3 秒（temporal attention window 有限）
- 编辑区域小且细节丰富（更容易看出帧间不一致）
- 同时编辑多个区域（追踪器压力大）

### 严重程度：★★★★

视频的帧间不一致是用户零容忍的问题——人眼对时序上的跳变极其敏感。一个 0.1 秒的闪烁就够破坏整个视频的"真实感"。在抖音场景下，用户发出去的视频如果有明显闪烁，会被观众当成"粗制滥造的 AI 痕迹"。

### DiT 时代是否缓解

**有进展但仍是开放问题。** Video DiT 模型（如 CogVideoX、Sora）在原生生成时帧间一致性好，但在"编辑"场景下的一致性优化公开研究较少。可灵和即梦的视频局部重绘功能在持续迭代中（待验证具体方案），Runway 的 Gen-3 video inpainting 在 2025 年有较好的效果反馈，但详细技术细节未公开。

## 问题之间的关联关系

```
P1 联动漂移 ←→ P4 多轮累积漂移
    │                    ↑
    │            每轮的 P1 累积成 P4
    │
    ├── 为了避免 P1 把参数调保守 → 触发 P2 欠编辑
    │
    └── 为了避免 P2 把参数调激进 → 触发 P3 过编辑

P5 Mask 边界 ←→ P1 联动漂移
    为了避免 P1 用 inpainting 硬锁 → 但引入 P5 边界问题

P6 帧间不一致 = P1 + P5 在时间维度上的放大版
```

**核心 tradeoff**：P1（联动漂移）和 P2（欠编辑）是一个光谱的两端——不可能同时完美。所有产品的编辑功能本质上都在这个光谱上选一个平衡点，或者给用户一个旋钮让他们自己选。

## 不同技术路线下各问题的严重程度

| 问题 | Inpainting | P2P | InstructPix2Pix | FLUX Kontext / Step1X-Edit | Seed+img2img | 视频编辑 |
|------|:---:|:---:|:---:|:---:|:---:|:---:|
| P1 联动漂移 | 低（mask外硬保） | 中 | 高 | 中低 | 高 | 高 |
| P2 欠编辑 | 低 | 中（τ太大时） | 中 | 低 | 高（strength低时） | 中 |
| P3 过编辑 | 中（mask内自由度大） | 低 | 中 | 中 | 中 | 中 |
| P4 多轮漂移 | 中（每轮要重画mask） | 高 | 高 | 低（专门优化） | 高 | 极高 |
| P5 边界不自然 | 高（核心短板） | 无（无mask） | 无 | 无 | 无 | 高 |
| P6 帧间不一致 | — | — | — | — | — | 高（核心短板） |

## DiT 时代的缓解总结

FLUX Kontext 和 Step1X-Edit 代表的 DiT 编辑范式对上述问题的缓解分布不均：

| 问题 | 缓解程度 | 原因 |
|------|----------|------|
| P1 联动漂移 | ★★★★ | Token concat 让模型更好地"看见"原图，保留能力更强 |
| P2 欠编辑 | ★★★ | 更强的文本编码器（T5-XXL / Qwen-VL）提升指令理解 |
| P3 过编辑 | ★★ | 没有专门的力度控制机制，改善有限 |
| P4 多轮漂移 | ★★★★ | Kontext 专门优化，但"minimal"不等于"zero" |
| P5 边界不自然 | ★★★★ | FLUX.1 Fill 等 inpainting 专用模型边界质量大幅提升 |
| P6 帧间不一致 | ★★ | Video DiT 在原生生成好，编辑场景仍是开放问题 |

**结论**：DiT 时代最大的突破是 P1（联动漂移）和 P4（多轮漂移）——这两个恰好是用户体验的最大痛点。P3（力度控制）和 P6（视频一致性）仍是 2025-2026 年的行业难题。

---

## 对产品设计的启示

1. **P1 和 P4 是用户流失的主因**——产品层要用"版本历史 + 分支回退"兜底，不依赖模型完美。

2. **P2 和 P3 是同一个旋钮的两端**——产品应该给用户一个"编辑强度"滑块（即使底层映射到不同参数组合），让用户自己调整平衡点。

3. **P5 的解法已经比较成熟**（DiT inpainting + auto mask），抖音侧可以直接用 MLLM 自动分割 + FLUX Fill 级方案作为默认体验。

4. **P6 在当前算力条件下建议产品层规避**——视频迭代编辑不作为核心功能推，或限制在"一键变体"而非"多轮精修"。

5. **手机端 mask 精度是被严重低估的问题**——手指涂抹 mask 的精度远低于 PC 端鼠标。产品应该默认提供"AI 自动 mask"（点击即选中对象），手动 mask 作为高级选项。
